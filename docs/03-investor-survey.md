# 03. 투자자성향 설문 & 스코어링

Investor Agent의 입력 규격. MVP는 **룰 기반 점수화**로 시작하고, 이후 LLM이 자유서술을 보조 해석하도록 확장한다.

## 1. 설문 문항 (MVP 최소 세트)

| ID | 문항 | 유형 | 스코어 반영 |
| -- | -- | -- | -- |
| Q1 | 나이 | 수치 | 기간·리스크 |
| Q2 | 투자 기간 | 1(<3년) ~ 5(10년+) | 리스크 ↑ |
| Q3 | 투자 목적 | 보전 / 균형 / 증식 | 리스크 |
| Q4 | 투자금 규모 | 수치 | 제약 |
| Q5 | 월 적립 가능액 | 수치 | 현금흐름 |
| Q6 | 최대 손실 감내(1년) | -5% / -10% / -15% / -25% / -35%+ | **핵심 제약** |
| Q7 | 투자 경험 | 없음 / 보통 / 많음 | 리스크 |
| Q8 | 원화 지출(단기 현금 필요) | 낮음 / 보통 / 높음 | 유동성·환노출 제약 |
| Q9 | 환노출 선호 | 회피 / 일부허용 / 허용 | 환헤지 정책 |
| Q10 | 손실 발생 시 행동 | 매도 / 유지 / 추가매수 | 리스크(행동) |

## 2. 스코어링 (룰 기반 예시)

각 문항을 0~4점으로 환산, 가중 합산해 **위험점수(0~100)** 산출.

```text
위험점수 = 0.30*Q6점수 + 0.20*Q2점수 + 0.15*Q10점수
         + 0.15*Q3점수 + 0.10*Q7점수 + 0.10*Q1점수(역)
```

> Q6(손실 감내)이 최고 가중치 — 손실감내도는 **하드 상한**으로도 동작(아래 4번).

## 3. 위험점수 → 성향 → 모델포트폴리오

| 위험점수 | 성향 | 모델포트폴리오 | 주식 비중 |
| -- | -- | -- | -- |
| 0–25 | 안정형 | MP-Conservative | 30% |
| 26–50 | 중립형 | MP-Neutral | 55% |
| 51–75 | 성장형 | MP-Growth | 75% |
| 76–100 | 공격형 | MP-Aggressive | 90% |

(비중은 상위 기획안 M7~M9 표와 일치)

## 4. 하드 가드레일 (Compliance 입력)

점수와 무관하게 아래는 강제 적용:

> **부호 규약(2026-07-03 R2 정정)**: Q6 선택지 `-5% / -10% / -15% / -25% / -35%`는 **음수일수록 손실 감내가 큰(공격적)** 응답이다. 과거 "-10% 이하" 같은 부등호 표기는 의도(보전 성향)와 정반대로 구현될 위험이 있어(예: `q6 <= -0.10`이면 가장 공격적인 -35% 응답자가 캡되고 -5% 최보수 응답자가 빠져나감), **집합 열거로 고정**한다. "예상손실 > Q6 초과"는 "예상손실이 Q6보다 더 음수"를 뜻한다(감내 한도 위반).

- **Q6 ∈ {-5%, -10%}(보전 성향)이면 → 성향 등급 상한을 중립형으로 캡. 그중 Q6 = -5%(최보수)면 → 안정형으로 캡.**
- Q8이 "높음"(단기 현금 필요)이면 → 현금+단기채 최소 비중 상향(07 §3: `CASH_ST` ≥ 10%)
- Q9가 "회피"면 → 환노출 자산에 환헤지형 우선 매핑(**MVP는 헤지 슬롯 미제공 → IPS에 "환헤지 미지원·통화노출 %" 정성 고지, 05 §1.8**)
- **어떤 성향이든** 예상 연손실이 Q6 감내 한도를 초과하면(더 음수이면) → **한 등급 강등 후 재계산** (05 §3 "차단→강등→재계산"과 정합. 전 등급 적용 — 과거 "공격형 시" 한정 표기는 05와 불일치라 정정)
- 강등 종료조건: 최대 3회 강등 후에도 위반 잔존 시 → **안정형 확정이 아니라, 안정형에서도 감내 한도 미달이면 "적합 상품 없음 — 배정 보류"**를 정식 종료상태로(포트폴리오 미발행, `breaches`에 사유 기록). 06 §2·08 §4 재계산 루프 종료조건.

## 5. 출력 스키마 (Investor Agent → IPS Agent)

```json
{
  "risk_score": 62,
  "profile": "성장형",
  "horizon_years": 8,
  "max_annual_loss": -0.20,
  "liquidity_need": "낮음",
  "fx_preference": "일부허용",
  "constraints": {
    "min_cash": 0.05,
    "hedge_preferred": false,
    "cap_profile": null
  }
}
```

> **constraints는 자유텍스트가 아니라 구조화 객체(2026-07-03 R2)**. allocation·compliance가 결정론적으로 소비하려면 `["현금 최소 5%"]` 같은 자연어가 아니라 `{min_cash: 0.05, ...}`여야 한다. `cap_profile`은 하드 가드레일(§4)이 성향 상한을 캡했을 때의 상한 등급(`"중립형"`/`"안정형"`/`null`). 강등 시 compliance는 이 객체를 **보존한 채** `RevisedProfile`을 재발행한다(constraints가 강등에서 증발하지 않도록, 06 계약).
