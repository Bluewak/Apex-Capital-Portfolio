## 2. CLI 설계 (프론트 = 명령줄)

콘솔스크립트 `apex`. 각 단계는 독립 실행 가능하고, `apex run`이 전체를 관통한다(06 §6).

```text
apex run --input answers.json [--currency krw] [--out report.html]
    # E2E: 설문→성향→배분→백테스트→리스크→컴플라이언스(강등 루프)→IPS→리포트 (수용기준 ≤3분)

apex survey score --input answers.json          # → InvestorProfile(JSON)
apex portfolio build --profile profile.json     # → Allocation
apex backtest run --alloc alloc.json --years 20 # → BacktestResult
apex risk report --returns series.parquet       # → RiskReport
apex comply check --risk risk.json --profile p.json  # → 위반 판정·강등(RevisedProfile)·재계산 트리거
apex ips render --profile p.json --alloc a.json # → IPS (compliance 확정 후)
apex data pull [--years 20]                      # 코어 9슬롯(+AGG)+FX+금리 raw 스냅샷 갱신
```

- 입력: 설문 응답 JSON(06 §3.1 스키마)이 **엔진 계약**. **단, 타깃(D2 국내 개인·첫 투자자)은 CLI/JSON로 진입 불가 → "사용자가 닿는 MVP"의 최소 조건으로 설문 웹폼·리포트 뷰어를 M6 범위로 승격**(R4 결정, §9). 엔진은 CLI로 먼저 증명(M4~M5), 진입 표면은 M6.
- 출력: 단계별 JSON 아티팩트 + 최종 HTML 리포트. 모든 산출물에 입력스냅샷·모델버전·**as-of 데이터 시점(`data_version`)** 기록(재현성 대면 표기, 05 §0·§9).

## 3. 마일스톤 (2026-07-03 레드팀 R2→R3 재배치: 척추→정확성→폭→경화)

> **재배치 원리**: 넓은 병렬을 먼저 벌이지 않는다. **① 계약(양 간선 포함)을 코드로 동결 → ② 얇은 수직 슬라이스(walking skeleton)로 재현성·강등 루프 종료를 실 20년 1케이스로 선증명(M4) → ③ 조정 정확성 대사(M4.5) → ④ 폭(4포트·지표, M5) → ⑤ 랜덤 전수 검증(M6)**. "성향 위반 0건" 랜덤 property test는 전 루프를 실데이터로 돌려야 하므로 **M5 DoD가 아니라 M6 E2E**다. **M4는 조정 "정확성"을 크리티컬 패스에서 빼고(피닝 조정종가 사용) 플럼빙·계약·재현성만 증명**한다(R3).

### M4 — 척추(Spine): 계약 동결 + 데이터 스냅샷 + Walking Skeleton  ✅ 구현 완료(2026-07-07, `c0d690f`)
- **schemas 동결** — pydantic 밖으로 새는 두 간선을 **모두** 1급 계약으로 닫는다:
  - **역간선(compliance→allocation)**: 반환 = `{decision, revised_profile: InvestorProfile|None, breaches}`, `decision ∈ {ok, downgrade, hold}`. 강등은 `downgrade(profile)→InvestorProfile`(constraints 보존, 03 §5) 한 곳에서. 문자열 `"중립형"` 반환 금지. **`downgrade()`는 allocation이 키로 소비하는 `profile`/`cap_profile`을 낮춰야 함 — `risk_score`(62 고정)를 낮추면 강등이 no-op**(R3 아키텍트).
  - **ReturnSeries 파케이 계약**(backtest→risk out-of-band): index=거래일 캘린더 tz-aware `DatetimeIndex`, 정확히 1개 `float64` 컬럼 `ret`(05 §0 단순수익률), parquet 메타에 `currency`+`data_version` 태그, **NaN 금지**(fill은 게이트①). backtest 출력·risk 입력이 **로드 시 이 계약을 검증**(위반=하드 실패). 06 §3.4 `returns_daily_ref` 불투명 핸들 대체.
  - **hold 렌더 계약**: `decision="hold"`(배정 보류) 시 report/ips가 낼 산출물 = 보류 사유 + 전문가 상담 권유. **대체 포트폴리오 노출 금지**(R3 자문 — 금지 안 하면 "알고도 발행"이 뒷문 복귀). null-allocation 분기의 exit code 정의.
- **데이터 스냅샷 동결**(재현성 토대, §6): 코어 **9슬롯**(SPY·QQQ·EFA·EEM·IEF·TLT·**AGG**·SHY·GLD) **+ 벤치마크 KODEX200(`069500.KS`)**을 `auto_adjust=False` **raw OHLCV + 배당/분할 이벤트**로 수집 → 정규화 raw에 content-hash(=`data_version`) 피닝. USD/KRW(FRED `DEXKOUS`)·무위험금리 수집. **skeleton이 쓸 조정종가는 M4에서 1회 스냅샷 피닝해 그대로 사용**(로컬 조정 엔진은 M4.5로 분리 — 조정 정확성 난제를 척추 크리티컬 패스에서 제거, R3 PM·데이터).
- **M4 데이터 게이트 3개**(최소): ① 거래일 캘린더(pandas-market-calendars) 커버리지 + fill 정책 — **정본 캘린더=XNYS(9슬롯), 커버리지 ≥99.9%·연속결측 ≤N일·fill 상한을 값으로 명시** ② 정의된 sanity 스위트(단조 날짜·음수/0 없음·분배/분할일 스파이크 검출) ③ raw 스냅샷 content-hash. (자동 소스 다중화·상시 Stooq 대사는 컷 — Stooq는 1회성 수동 백필만.)
- **Walking skeleton**: 1개 프로파일(**강등을 강제하는 적대적 케이스** — 공격형 요청인데 평시 상한 초과)로 investor→allocation→backtest(CAGR·MDD)→risk→compliance(루프)→문장 1줄→해시를 실 20년으로 **관통**. **강등 경로상 등급들의 고정비중 벡터를 M4 산출물로 명시**(4종 전체는 M5, 경로 등급만 M4 — 스텁 가중치면 역간선이 no-op가 되어 DoD(b) 무의미해짐, R3 아키텍트).
- **DoD**: (a) skeleton E2E 1케이스가 **동일 스냅샷 재실행 시 산출 수치필드 rtol≤1e-6 일치**(재현성, §6) (b) **강등 루프가 실제로 돌고 종료조건(수렴/hold)에서 멈춤**(역간선 실증) (c) 9슬롯+벤치마크×20년 raw 스냅샷이 게이트 3종 통과.

### M4.5 — 조정 정확성(Adjustment Accuracy): raw→로컬 TR 엔진 + 대사 오라클  ✅ 구현 완료(2026-07-07, `6560060`)
> M4 skeleton은 피닝된 조정종가로 통과시켜 **플럼빙·루프·재현성**을 먼저 증명하고, "조정이 정확한가"는 여기서 별도로 닫는다(R3 PM·데이터 — 로컬 조정은 CRSP식 조정팩터 구현 = 서브프로젝트).
- **raw(unadjusted)+이벤트 → 로컬 결정론 총수익(TR) 조정 엔진** 구현(yfinance 소급조정 의존 제거).
- **⭐ TR 대사 오라클**: 로컬 조정 TR 시계열을 착수 1회 **독립 레퍼런스**(yfinance `auto_adjust=True` 스냅샷 또는 펀드 공시 NAV/TR)와 **연율 편차 ≤ 수 bp** 내 일치 검증. **9슬롯 + 벤치마크 전부**. (조정종가를 버리고 Stooq 대사도 컷했으므로, 이 대사가 로컬 조정을 교차검증할 유일한 오라클 — 없으면 content-hash가 "재현 가능하게 틀림"만 보증, R3 데이터.)
- **DoD**: 전 슬롯·벤치마크가 대사 허용오차 통과. 이후 backtest는 피닝 조정종가 → 로컬 조정 TR로 소스 전환.

### M5 — 폭(Breadth): 고정포트 5종 + 포트↔상한 사전검증  ✅ 게이트 구현 완료(2026-07-07, `b08720b` + 튜닝 `90ae2e4`)
- allocation: 성향별 고정비중 **5종(초안정/안정/중립/성장/공격)**, 티커 매핑(02·07 §3). **안정형은 AGG 채권 코어, 초안정형은 near-cash(SHY 중심) 반영**(R5, §10).
- **⭐ 07 §7 포트↔상한 사전검증 게이트**: 5종 포트가 각 성향의 05 §3 **평시 상한**(vol/MDD/연율VaR, R3: 위기 3구간 제외)을 실측 통과하는지 확정. **초안정형은 `VaR95_annual ≤ -5%` 실측 통과가 발행 조건**(미통과 시 발행 금지 → 예금 referral; 미검증 라벨=배신, R5). 2008/2020/2022 실측 손실은 **차단이 아니라 리포트 disclosed 스트레스로 고지**(R3 "스트레스는 공시, 차단은 평시"). 평시 상한도 미통과인 포트만 무게 조정 or 상한 개정 — **개정 폭 상한을 미리 못박아 루프 유한화**(R3 PM).
- backtest: 일별수익률·CAGR·MDD, **벤치마크 = ETF TR 프록시**(S&P500→SPY 조정종가 / 60·40→SPY60+IEF40 / KOSPI200→KODEX200 `069500.KS`). `^GSPC`/`^KS200`(price-return) 금지. **삼중 캘린더 정렬 규칙 명시**(9슬롯=XNYS / KODEX200=XKRX / DEXKOUS=FRB → 공통 거래일 inner join, FX as-of forward-fill). **Sharpe 무위험은 각 시계열의 계산통화 기준**(포트=USD 3M, KODEX200=KRW-CD)으로 산출 후 위험조정 비교(통화 불일치 왜곡 방지, R3 퀀트·데이터). **상수 확정**: 거래비용 = 0(리포트 명시) 또는 편도 5bps, 리밸런싱 = 분기말, 환산 = daily.
- risk: **축소 지표세트**(vol/MDD/**Historical VaR95 1d+`var95_annual`(05 §1.3 단일식)**/CVaR95/Sharpe/Calmar/집중도 + 통화노출%). 금리·주식충격 민감도, 가상 스트레스 4종, 통화 2종 저장은 **v2**.
- compliance: 05 §3 **평시** 상한 대비 차단→강등→재계산 루프(종료조건 03 §4, hold 포함). `apex comply` 동사.
- **DoD**: 5종 포트가 각 성향 **평시** 상한을 통과함을 확인(**초안정형 VaR95_annual ≤-5% 실측 통과 포함**) + 실측 스트레스가 공시로 출력됨 + 05 §5 체크리스트.

### M6 — 경화(Hardening): 리포트 + 진입 표면 + 교육 계층 + 랜덤 전수  🟡 다음 단계
- IPS 문장 템플릿(04 §2) + 룰기반 설명 리포트(추천마다 자연어 1회 이상, "추천" 단위 정의 명시).
- **규제 하드요구**(§4-5): 면책·투자권유 아님·원금손실·과거성과 무보장 고지 렌더 + **"예시 유형 귀속형" 프레이밍**(개인 지시형 금지) + 개인식별정보 배제(Q4/Q5 밴드) + 사용약관 "유형별 교육 정보·개별 자문 아님" 동의 게이트.
- **진입 표면(R4)**: 최소 설문 웹폼(10문항, Q9 등 용어 툴팁) + HTML 리포트 뷰어. CLI/JSON은 엔진용으로 유지.

## 7. 의존 순서 (병렬 착수 지도)

```text
schemas ─▶ data ─▶ ┌─────────── 재계산 루프 ───────────┐
                   │                                   ▼
   investor ─▶ allocation ─▶ backtest ─▶ risk ─▶ compliance ─(위반: 강등)─┐
                   ▲──────────────────────────────────────────────────────┘
                                                              │(수렴)
                                                              ▼
                                                  ips ─▶ report ─▶ pipeline(apex run)
```
- **이건 DAG가 아니라 사이클**이다: `compliance ─(위반)─▶ allocation` 역간선(강등 재배분)이 있다(06 §2). 이 간선은 `RevisedProfile` 1급 계약(03 §5)으로 서고, **루프 소유는 pipeline**(`while decision==downgrade and n<3: profile=revised; …; else: hold`). 종료조건: 수렴 or 3회 후 `decision="hold"`(배정 보류, 03 §4).
- **`downgrade()`는 allocation이 키로 읽는 값(`profile`/`cap_profile`)을 낮춰야** 강등이 실효한다. `risk_score`(예: 62)를 그대로 두고 등급 문자열만 바꾸거나, allocation이 `risk_score`를 키로 소비하면 **강등이 no-op**가 되어 루프가 헛돈다(R3 아키텍트).
- **null-allocation(hold) 분기**: `decision="hold"`면 alloc=∅ → `apex ips render`/report는 SAA·밴드를 채울 수 없다. 이 경우 **hold 전용 산출물**(사유+상담 권유+교육용 대안, 개인화 대체 포트 노출 금지)을 렌더하고 정의된 exit code를 반환한다. "모든 추천 자연어 100%"·≤3분·리포트 계약은 이 무배정 경로를 포함해 닫는다(R3 아키텍트).
- **downgrade 분기(R4)**: `decision="downgrade"`로 등급이 낮아진 경우 report는 **강등 사유를 사용자 언어로 렌더**(03 §4)해야 한다 — hold에만 문안을 두고 강등에 안 두면 "버그" 오인·Q6 게이밍을 부른다(R4 CS 티켓5).
- **IPS는 compliance 확정 후**에 렌더한다(강등 뒤 최종 SAA·허용손실이 정해지므로 — investor 직후 렌더 금지).
- schemas·데이터 스냅샷 동결 + skeleton(강등 강제 1케이스) 통과 = **M4 완료 후에만** allocation/backtest/risk의 폭 확장을 병렬 착수. 계약 미실행 상태의 "종이 병렬"은 순환 은닉이므로 금지.

2. **이해도 레이어**: IPS·SAA·MDD·VaR·Sharpe·CAGR 용어집·툴팁(progressive disclosure). DoD에 "비전문가 핵심 3개념 이해" 사용성 테스트.
3. **공포 숫자 프레이밍**: disclosed 스트레스(−18% 등) 숫자 옆에 **안심 내러티브를 라벨만큼의 비중**으로(단독 노출 금지).
4. **응답→산출 연결**: 투자자 응답이 산출에 반영된 흔적 렌더(04 §2 `{reflected_answers}`) — "물어보고 안 지킴" 배신감 방지.
5. **화폐환상 대응**: 환효과 분리(자산손익 vs 환손익)를 **환회피 응답자 기본 표기**(선택 아님).
6. **행동 개입**: Q10(손실 시 행동)을 수집만 말고 **하락기 행동 대본·사전서약**으로 활용(04 §1).
7. **재현성 대면 번역**: 리포트에 as-of 데이터 시점·`data_version` 표기 + 재실행 변동 FAQ(“사기” 오인 방지, CS 티켓2).
8. **강등 사유 렌더 + 게이밍 억제**: downgrade 사유 사용자 언어 렌더 + Q6 급변 경고(03 §4).
9. **CS 런북**: 티켓 분류·에스컬레이션·disclosed FAQ·환불/이의 정책(개발계획 밖이던 운영 문서 신설).

**신규 KPI(재현성·정확성 외 — 08은 지금까지 엔진 지표만 있었음)**: 설문 완주율, 초보 이해도(사용성), hold 후 이탈률/대안 열람률, 재실행 변동 문의율.

## 10. 핵심 보수 세그먼트 & hold 해체 (2026-07-05 레드팀 R5)

손 시뮬(§ artifact)이 드러낸 문제: **1순위 타깃(은퇴·보수, Q6 감내 -5%)의 기본 경로가 사실상 배정 보류(hold)**다. binding=min(안정형 -8%, Q6 -5%)=-5%인데 최하위 안정형(주식30)조차 평시 예상손실 ≈-8%라 위반 → 강등 불가 → hold. 전략가·퀀트·컴플·행동재무 4인 토론 결론: **코드 버그가 아니라 (1) 저해상도 사다리 (2) 측정 오류 (3) 잘못된 바인딩 지표의 3중 문제.** 아래 **3단 해법**으로 hold를 기본값에서 진짜 엣지케이스로 해체한다.

**1단 — 재보정 관문 (hold 이전, 03 §1·§4 연동)**: "감내 몇 %?" 추상 질문을 **결과 결합형**("6년에 한 번 -15% vs 예금이지만 실질 구매력 하락")으로 + **인플레이션을 '손실'로 재프레이밍**("예금도 실질 -Z% 손실"). Q6=-5% ∧ 목표=물가+α 같은 **모순 주문**을 감지하면 hold 발행 전에 갈림길 인터랙션("무손실 우선? 물가+α 우선?"). → 방어적 -5%는 완화(→안정형), 진짜 -5%만 남긴다. **교육(트레이드오프 곡선)이지 개인 지시 아님** — 맞춤성 경계 안 넘음.

**2단 — 초안정형(near-cash) 티어 신설 (05·07·03 파장)**: 진짜 -5% 고객에게 near-cash **유형 예시**. 강등 사다리 **안정형→초안정형→hold**로 즉시-hold 해소.
- **바인딩 지표 = `VaR95_annual`로 확정**(MDD 아님). 구성 예시(SHY65/SPY8/GLD8/현금): σ≈1.9% → VaR95≈3.1% → -5% 안. **MDD -5%는 20년·무splice에서 물리적 불가**(SHY 2022 -5.5%)라 초안정형 **평시 MDD 상한 ~-7%** + 2022는 disclosed(§R3). near-cash slot=SHY(BIL/SGOV splice는 v2).
- **몰개성 "유형 예시"만**(개인화 배분 발행 금지 — 자문 경계) + IPS **트레이드오프 고지**(기대 ~연 2.5%, 인플레 미달 가능 = informed consent). 미검증 라벨=배신 → **M5 §7 게이트에서 평시 VaR≤-5% 실측 통과가 발행 조건**.

**3단 — 정직한 hold를 히어로 플로우로 (M6)**: 재보정·초안정형에도 안 걸리는 진짜 hold(모순 주문 / 명목 무손실 지상)는 **예금·MMF referral + "왜 안 되고 대신 뭐가 맞는지"를 완결된 제품으로**. hold 화면 = M6 corner case가 아니라 **주력 인수기준**(실 은퇴자 페르소나 사용성 테스트). *로보들이 전부 억지로 포트에 밀어넣을 때 우리만 "안 사도 된다"를 말하는 신뢰 브랜드.*

**규제 쐐기(컴플)**: "포트 안 줬으니 안전"은 착각 — 개인화된 hold("현금성으로 가세요")도 개별 자문 경계. hold·초안정형 공통 안전판 = **몰개성 유형 예시 + 검증된 손실한도 + 정직한 트레이드오프** 3종을 동일 강제. 등록 전 상담은 referral로만.

**GTM 재프레임(전략가, R5 결정)**: 타깃 재설정은 거부하되 **"수익 타깃 ≠ 신뢰 타깃" 분리** — 중립~성장 = v1 매출(유료 리포트·심화), **보수 세그먼트 = 신뢰·리드 퍼널**(정직한 "예금이 맞다" → 최고 신뢰 리드 → 자문 등록 후 상담·재유입). 보수층용 유료 기능에 로드맵 태우지 말 것. 보수층 북극성 = 포트 발행이 아니라 **hold 후 이탈률↓·대안 열람률↑·재유입율**(§9 KPI를 주력으로).
